# Snip — утилита для скриншотов (Linux/Wayland/GNOME)

## Обзор

Десктопная утилита для захвата области экрана с встроенным редактором аннотаций. Написана на Rust + GTK4/Libadwaita. Целевая платформа — Linux, Wayland, GNOME последних версий.

## CLI-интерфейс

```
snip              — захват области экрана → сохранение → редактор
snip edit <path>  — открыть существующий файл в редакторе
```

Привязка к Print Screen выполняется пользователем вручную через настройки GNOME (custom keybinding → команда `snip`).

## Поток захвата (Capture Flow)

1. Пользователь запускает `snip`
2. D-Bus вызов `org.freedesktop.portal.Screenshot` — получаем полный скриншот текущего монитора во временный файл
3. Открывается полноэкранное окно-overlay (без декораций, поверх всех окон):
   - Фон — полный скриншот экрана
   - Поверх — полупрозрачный чёрный слой (rgba 0, 0, 0, 0.4)
   - Курсор — crosshair
4. Пользователь нажимает ЛКМ и тянет — выделяет прямоугольную область:
   - Выделенная область отображается без затемнения (вырез в overlay)
   - Тонкая белая рамка (1-2px) вокруг выделения
5. Отпускание ЛКМ — захват завершён:
   - Изображение обрезается по выделенной области
   - Сохраняется в файл (см. раздел "Сохранение")
   - Overlay закрывается, открывается окно редактора
6. Escape в любой момент — отмена, приложение завершается

### Ограничения overlay

- Выделение меньше 5×5 px игнорируется (защита от случайного клика)
- Мультимонитор: portal возвращает скриншот текущего монитора — для MVP достаточно

## Редактор (Editor)

### Компоновка окна

```
┌─────────────────────────────────────────────────┐
│ HeaderBar                                       │
│ [🖌 Кисть] [+ ...]     [Копировать] [Путь] [💾] │
├─────────────────────────────────────────────────┤
│                                                 │
│                  Canvas                         │
│          (скриншот + аннотации)                  │
│                                                 │
├─────────────────────────────────────────────────┤
│ Цвет: ● ● ● ● ● 🎨  │  Толщина: • ● ⬤          │
└─────────────────────────────────────────────────┘
```

- **HeaderBar** (Adwaita): инструменты слева, кнопки действий справа
- **Canvas**: центральная область, скриншот отрисовывается через Cairo, аннотации — поверх на отдельном слое
- **Нижняя панель**: параметры текущего инструмента

### HeaderBar — кнопки действий

| Кнопка | Действие |
|--------|----------|
| Копировать | Копирует текущее состояние (оригинал + аннотации) в системный буфер обмена через GTK4 clipboard API |
| Путь | Копирует абсолютный путь к файлу как текст в буфер обмена |
| Сохранить | Перезаписывает файл — оригинал + аннотации "впекаются" в PNG |

### Инструмент "Кисть"

Первый и единственный инструмент в MVP. Свободное рисование линий поверх скриншота.

**Параметры (нижняя панель):**
- Цвет: преднастроенные варианты (красный, зелёный, жёлтый, синий, розовый) + произвольный выбор через GtkColorDialog
- Толщина: 3 варианта (тонкая / средняя / толстая)

**Поведение:**
- Mouse down — начало stroke
- Mouse move — добавление точек к текущему stroke
- Mouse up — завершение stroke, добавление в список аннотаций

**Модель данных stroke:**
```rust
struct Stroke {
    points: Vec<(f64, f64)>,
    color: gdk::RGBA,
    width: f64,
}
```

### Расширяемость инструментов

Архитектура через trait:

```rust
trait Tool {
    fn on_press(&mut self, x: f64, y: f64);
    fn on_move(&mut self, x: f64, y: f64);
    fn on_release(&mut self, x: f64, y: f64);
    fn draw(&self, cr: &cairo::Context);
}
```

Редактор хранит `Vec<Box<dyn Tool>>` завершённых аннотаций + текущий активный инструмент. Добавление нового инструмента = новый файл в `src/tools/`, реализация trait, регистрация в HeaderBar.

## Сохранение файлов

### Путь сохранения

`XDG_PICTURES_DIR/Screenshots/screenshot-YYYY-MM-DD_HH-MM-SS.png`

- `XDG_PICTURES_DIR` определяется через `xdg-user-dirs` (crate `dirs` или `xdg`)
- Подпапка `Screenshots/` создаётся автоматически если не существует
- Формат: PNG

### Именование

`screenshot-YYYY-MM-DD_HH-MM-SS.png` — timestamp момента захвата.

### Автосохранение

Скриншот сохраняется автоматически сразу после crop, до открытия редактора. Кнопка "Сохранить" в редакторе перезаписывает этот же файл с аннотациями.

## Структура проекта

```
snip/
  Cargo.toml
  src/
    main.rs          — CLI-парсинг (clap), запуск GtkApplication
    capture.rs       — portal screenshot + overlay selection
    editor.rs        — окно редактора
    tools/
      mod.rs         — trait Tool + реестр инструментов
      brush.rs       — инструмент кисти
    utils.rs         — сохранение файлов, XDG paths, буфер обмена
```

## Зависимости (crates)

| Crate | Назначение |
|-------|-----------|
| `gtk4` | UI framework |
| `libadwaita` | GNOME Adwaita стиль |
| `ashpd` | XDG Desktop Portals (screenshot) |
| `cairo-rs` | Рисование (overlay, аннотации) |
| `clap` | CLI-парсинг |
| `chrono` | Timestamp для имён файлов |
| `dirs` | XDG directories |

## Обработка ошибок

Минимальная, только реальные сценарии:

- Portal отклонён пользователем → тихое завершение
- Выделение < 5×5 px → игнорируется
- Папка Screenshots не существует → создаётся
- `snip edit` с несуществующим файлом → ошибка в stderr, exit code 1
- Escape в overlay → отмена, выход

Никаких диалогов с ошибками, retry-логики, fallback-стратегий.
